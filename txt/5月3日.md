## 二、关键架构变更

### Step 1: `--from-cache-only` 全量离线重建模式

- 新增 `main.py --from-cache-only`：仅从 911 个缓存 HTML 重建，零网络请求
- 全量遍历替代随机采样，消除 `--limit` 造成的覆盖不足
- 输出统计：总缓存 / 解析成功 / 变体展开 / 去重 / 最终物品数

### Step 2: 过滤策略松弛

- `is_reject_page()` 从 40+ 关键词缩减到仅 3 类：
  1. April Fools / 玩笑内容
  2. 消歧义页面
  3. 命令 / 教程页
- 新增实体/概念页检测（Entities/Mobs 无 Items/Blocks 分类 → 拒绝）
- 移除：historical、bedrock edition、unused、removed 等过宽拦截

### Step 3: 变体展开引擎（核心突破）

**新建 `variant_expander.py`（439 行）**

从概览页提取个体变体：
- **Infob​​ox BlockSprite 解析**：`BlockSprite_{color}-wool.png` → 16 色羊毛
- **Invicon 解析**：`Invicon White Carpet.png` → 白色地毯
- **全页内容回退**：Boat 页图片不在 infobox，扫描全文
- **Spawn Egg 专用解析**：从 wikitable 提取 82 个实体

### Step 4: 图标 URL 全量修复

- `extract_icon_url` 重写：搜索范围从仅 infobox → 全文三阶段扫描
- `normalize_icon` 重写：保留 CDN hash 目录，正确处理 thumb/archive 路径
- Sprite URL 自动替换为 `Special:FilePath` 重定向（302 → 真实图片）

|----------|--------|--------|
| 真实 CDN 图片 | ~227 | **288** |
| Special:FilePath | 279 | **519** |
| BlockSprite/Invicon | ~300 | **0** |

### Step 5: 分类修正 + 中文补全 + 翻译修复

**分类修正（197 项）**：
- `normalize_category()` 基于 ID 模式：`*_wool` → colored_blocks，`*_spawn_egg` → spawn_eggs 等
- 所有 `*_wool`、`*_planks`、`*_spawn_egg` 分类一致性通过

**中文补全（133 → 2 fallback）**：
- 新增 Layer 0 模式匹配：`get_zh_name` 优先基于规则生成中文名
- 阻止 "Chicken Spawn Egg" → 子串匹配 "Spawn Egg" → 只返回"刷怪蛋"的 bug
- `ZH_MAP` 扩充 ~80 条独有物品翻译
- 剩余 2 fallback：`Banners`、`Minecraft`（非物品页）

**翻译规则**：
```
{Entity} Spawn Egg → {实体中文}刷怪蛋
{Color} {Base}    → {颜色中文}{基类中文}
{Wood} {Base}     → {木种中文}{基类中文}
```
