## Step 3: 归一化逻辑

### normalizer.py — 中文覆盖 + 数据清洗
- ZH_MAP 从 ~350 条扩充到 **650+ 条**（+300 条翻译）
- `get_zh_name()` 从 3 层 → **5 层 fallback**：
  1. 直接查 ZH_MAP
  2. 清理 _ → 空格后查
  3. `_canonicalize_name()` 后查（大小写变体处理）
  4. 前缀子串匹配（Smooth Quartz Stairs → Smooth Quartz）
  5. 返回原文 title case
- `_canonicalize_name()` — 处理 `minecraft:` 前缀、`File:` 前缀、小写转大写
- `is_non_item()` — 29 个已知伪物品过滤（含 normalize_id 后的模糊匹配）
- `validate_quality()` — 对齐 plan.md §8.1 验收标准
- `validate_quality_report()` — 完整验证报告输出

### parser_list.py — 过滤器
- `is_reject_page()` — Wiki 分类黑名单过滤（April Fools / Education Edition / bedrock / historical / removed / joke features / commands）
- `get_subcategory_from_wiki_cats()` — 基于 Category Graph 规则的小类映射

## Step 4: 前端核心实现

### index.html — 骨架增强
- 3 组骨架屏：`#skeleton-categories`(8卡片) + `#skeleton-items`(12格子) + `#skeleton-detail`
- 4 个 `<template>` 元素：tpl-category-card / tpl-item-cell / tpl-suggestion-item / tpl-related-item
- ARIA 全覆盖：`role="search"` / `role="combobox"` / `role="dialog"` / `aria-expanded` / `aria-controls`
- JEI 级双向查询区：合成来源 + 可合成为

### search.js — 性能优化（800+ 物品 ≤ 100ms）
- **bigram 倒排索引**：数据加载时一次性构建，将候选集缩减至 20-40%
- **100ms 防抖**：避免高频输入触发全量搜索
- **结果缓存**：相同查询直接命中缓存
- **`<template>` 克隆渲染**：避免 innerHTML 字符串解析

### style.css — CSS Grid 3×3 配方 + 骨架屏
- Minecraft 风格槽位：`linear-gradient` 纹理 + `inset box-shadow` + hover 高亮环
- 空槽网格纹理：`repeating-linear-gradient(45deg, ...)`
- 数量角标描边：`text-shadow` 四向白色（Minecraft 风格）
- 骨架屏 shimmer 动画：`@keyframes skeleton-shimmer`
- JEI 主配方高亮：`related-item-primary` 绿色边框 + 角标
- 移动端适配：480px 断点 40px 格 + 320px 图标
